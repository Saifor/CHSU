MultiArc
~~~~~~~~
[!] Каждый FMT-модуль теперь может находится в отдельной папке
    каталога MultiArc\Formats.

[-] LZH.FMT не распознавал архивы, созданные более поздними версиями
    архиваторов LHA.

[-] CAB.FMT не корректно работал с некоторыми типами SFX-архивов,
    например ar500enu.exe - инсталляция Adobe Acrobat Reader 5.0

[!] CAB.FMT теперь имеет настройки по умолчанию, рассчитанные на работу с
    MsCab.exe версии 0.60b3 и выше. Последние версии архиватора можно взять
    здесь http://mateuszb.republika.pl/

[-] Многотомные CAB-архивы теперь распаковываются по ShiftF2 без "спотыкания"
    на "пустых" томах (начало файла в предыдущем томе, окончание в следующем)

[-] CUSTOM.FMT: исправлено поведение модификатора '*' в описании
    листинга архиватора. Ранее один символ обязательно обрабатывался,
    даже если был пробелом или символом конца строки.

[*] CUSTOM.FMT: теперь перехватываются сообщения архиваторов, выводимые
    архиватором в стандартный поток вывода ошибок.

[!] ACE.FMT корректно показывает размер словаря и поставляется с исходным
    текстом (см. PlugDoc.rar::Examples\MultiArc\Formats\ace.cpp).

[!] TAR.FMT и ZIP.FMT корректно работают с архивами, в которых имена файлов
    начинаются с префикса ".\"

[+] В архивах TAR добавлена поддержка расширения GNU, позволяющего
    хранить имена файлов длиннее 100 символов. Ранее такие имена
    показывались как файлы ".\.\@LongLink", видимые в каждой папке
    архива.

[+] Опция "Использовать последнюю команду из истории" - при добавлении файлов
    к архивам в строку "Ключи" будет помещено ранее используемое значение
    ключей командной строки архиваторов.

[!] Каждому типу архиватора соответствует своя история ключей для строки
    "Ключи" (диалог добавления файлов).

[+] Новая функция GetSFXPos для плагинов второго уровня. Возвращает позицию
    начала архива. MultiArc выбирает тот FMT-модуль, который вернул
    наименьшее значение (ближе к началу файла). Это исправляет ситуацию,
    когда в ZIP-файл с именем, например, "N3gk8v1t.106" (размер ~10K) помещен
    ARJ-архив без сжатия. И если в такой архив заглянуть предыдущими версиями
    MultiArc, то увидим содержимое именно ARJ-архива, а не ZIP.

[+] Новая опция в настройках конкретного архиватора - "Расширение архива".

[+] Новая утилита ma_emu.cpp, облегчающая отладку плагинов второго
    уровня (см. PlugDoc.rar::Examples\MultiArc\Formats\ma_emu.cpp)

[-] Неправильная обработка %%LW.

[+] Кнопка "Сохранить параметры" в диалоге "Добавить к архиву" теперь
    недоступна, когда нечего сохранять.

[+] Теперь в меню "Архивные команды" (ShiftF3) попадают только
    поддерживаемые архиватором команды.

[-] Архиваторы без поддержки файлов-списков (InfoZip) не могли нормально
    тестировать файлы, лежащие не в корне, им отдавались относительные
    пути вместо абсолютных.

[-] При добавлении файлов в архив с расширением, отличным от
    расширения по умолчанию, MultiArc больше не искажает имя
    архива. Создавать такие архивы можно используя опцию "Точное
    соответствие имени архива".

[+] В диалоге распаковки добавлена опция "Удалить из архива после распаковки"

[!] В диалоге добавления файлов клавиша Shift-F1 аналогична нажатию на кнопку
    "Архиватор" ("Select archiver").

[*] В командные строки RAR добавлен полезный ключ "{-ap%%R}".
    Это позволит паковать не только в корень архива
    и извлекать без создания всей структуры папок до текущей.

[*] Кардинально переработан алгоритм формирования имени архива, в следствии
    чего устранен целый букет ошибок.

[-] Теперь MultiArc корректно работает с именами архивов, содержащих точки.

[-] Имя архива не сбрасывается на имя по умолчанию при выборе архиватора.

[!] ВНИМАНИЕ! тем, у кого используется семейство архиваторов pkzip,
    настоятельно рекомендуется добавить во все команды ключ
    "-nozipextension". То же самое относится к другим архиваторам, которые
    сами дописывают расширение по умолчанию к имени вновь создаваемого
    архива - архиватор по возможности должен не вмешиваться в процесс
    формирования имени архива.

[*] Упразднено понижение регистра у строки имени архива по умолчанию
    в диалоге "Добавить к архиву" если она содержит буквы разных
    регистров.

[-] Исправлено ошибочное фоновое исполнение команд из меню ShiftF3.

[*] Cтрока в настройках архиватора, существующая в реестре, но пустая,
    более не заменяется строкой по умолчанию из fmt-модуля. Поскольку
    используемый архиватор (например - InfoZip) может не поддерживать
    всех команд (преобразование в SFX) архиватора по умолчанию
    (pkzip/pkunzip).

[*] Отключено мерцание строчкой "Архивная операция" при поиске в архивах.


ProcList
~~~~~~~~
[!] Внутренний список процессов обновляется асинхронно в специальной фоновой
    нити, так что плагин хорошо работает с удаленными компьютерами по
    медленной сети (например, по телефонной линии).

[+] Все режимы панелей переопределены для показа различной информации
    о процессах. Режимы настраиваются (пока только через реестр, см. хелп).

[+] В информацию, выводимую по F3, включен список модулей и хэндлов.
    В списке модулей можно также включить показ их версий и описаний,
    но по умолчанию он выключен.

[+] Под NT показывается больше счетчиков производительности (собственно,
    все счетчики).

[+] Новые пункты в конфигурации; Alt-Shift-F9 вызывает диалог конфигурации
    из панели плагина.

[+] Добавлены новые режимы сортировки (Ctrl-F12).

[+] Ctrl-F вставляет в командную строку полное имя с путем.

[+] Под Win9x/ME показывается PID родительского процесса.

[*] В строке состояния показываются "размер", дата и время процесса.

[*] История в диалоге выбора компьютеров.

[-] Падение FARа при попытке смотреть информацию о процессе из
    диалога результатов поиска (Alt-F7).

[-] Под Windows 98+ не работало переключение на окно другого процесса.

[-] На скрытые окна теперь нельзя переключаться; кроме того, видимые окна
    имеют преимущество перед невидимыми при показе в столбце описаний.

[-] Исправлены различные ошибки с перекодировкой букв.

[!] Теперь настройка разделителя между командой и параметрами может

TmpPanel
~~~~~~~~~
[!] Изменился синтаксис командной строки. Новый синтаксис не вызывает
    неоднозначности при открытии файлов-списков, название которых
    начинается с символов A или R.

[+] Новая опция в настройках, разрешающая добавление к меню плагинов
    пункта "Temporary panel".

[+] Если включена опция "Использовать общую панель", то плагин позволяет
    использовать до 10 панелей, каждая из которых содержит отдельный
    список файлов. Для переключения между панелями можно воспользоваться
    меню (комбинация Alt-Shift-F12) или клавишей для непосредственного
    переключения (от Alt-Shift-0 до Alt-Shift-9).

[+] Опция "Новая панель для рез-тов поиска" (действует при включенной
    опции "Использовать общую панель") позволяет поместить результаты
    поиска (Alt-F7 -> Панель) на свободную временную панель.
    Это позволяет сохранять результаты нескольких операций поиска
    и работать с ними одновременно.

[+] Комбинация Alt-Shift-F2 позволяет сохранить текущее содержимое
    панели в файл-список.

[+] Опция "Безопасный режим панели" делает недоступными все операции
    над элементами панели (копирование, перенос, удаление и т.д.),
    кроме перехода в папку и копирования в командную строку.

[+] Опция "Любые элементы в панели" отключает проверку на существование
    файла перед помещением его в панель. Нажатие Enter на строке, не
    соответствующей реальному имени файла, копирует ее в командную
    строку FAR.

[+] Опция "Меню из файла-списка". Если эта опция включена, при нажатии
    Enter на файле-списке вместо открытия панели показывается меню,
    содержащее все строки файла-списка.

[+] Добавлена возможность настройки типов и ширины колонок во временной
    панели и в строке статуса.

[+] Опция "Полноэкранный режим панели".

[+] Теперь поиск файла-списка производится не только в текущей папке,
    но и в папках, доступных через %PATH%.

[+] Теперь возможно поместить на временную панель список файлов,
    выдаваемых внешней программой на stdout.

[-] Неправильно обрабатывались файлы-списки с первой пустой строкой.

FARFtp
~~~~~~
[!] Расширенные опции, конфигурируемые в реестре Windows
    (см файлы "TechInfoFTP.Rus.txt")

[!] Операция и кнопка в диалоге перезаписи "resume all" (докачать все)
    (Кнопка "докачать" присутствует только если сервер поддерживает докачку.
    Новая строка в окне информации: возможность докачки с этого сервера.)

[!] Изменен механизм редактирования и использования данных записей.
    Имя пользователя и пароль имеют дополнительные строки редактирования в
    диалоге, и данные, занесенные туда, имеют больший приоритет, чем те,
    которые заданы в заголовке панели.

[!] Изменен механизм сохранения записей в реестре:
     - больше нет ситуаций, когда запись с совпадающим заголовком молча
       затирает старую.
     - нет ограничений на количество записей с одним и тем же заголовком
     - все существующие записи будут автоматически конвертированы в новый
       формат, как только пользователь подтвердит их изменение (выберет ОК при
       редактировании записи)
     - изменен/улучшен механизм экспорта записей
     - экспорт директории с описанием с панели плагина в файл приводит к
       потере описания

[!] Плагин не всегда работает в пассивном режиме.

[!] Любая операция может быть прервана нажатием на ESC в любое время.
    Прерывание операций по Ctrl+Break убрано вообще. Можно также
    включить запрос на подтверждение прерывания (см. TechInfoFTP.Rus.txt
    #4).

[+] Расширенное окно команд FTP. Размер окна настраивается (см.
    TechInfoFTP.Rus.txt #5, #6).

[+] Возможность ведения протокола выполненных команд и ответов
    FTP-сервера (см. TechInfoFTP.Rus.txt #8, #9, #16).

[+] Звуковой сигнал после длительных операций (см. TechInfoFTP.Rus.txt#11)

[+] Процедура генерации имени файла с панели FTP (имена, получающиеся по
    Ctrl+F) теперь генерирует полный URL, который содержит всю информацию о
    текущем файле, включая имя пользователя и пароль

[+] Описания могут быть введены и для директорий на панели записей (в диалоге
    создания директории есть новое поле, которое недоступно, если создается
    директория на FTP). Это описание можно добавить/редактировать, нажав F4 на
    имени каталога

[+] Две новые колонки в режиме записей: путь и имя пользователя

[+] Добавлено рисование скопированного объема текущего и общего в диалоге
    копирования (может быть выключено правилом TechInfoFTP.Rus.txt#18)

[*] Возможность настройки разделителя тысяч в диалоге копирования
    (см. TechInfoFTP.Rus.txt #1, #2). Разделитель тысяч, установленный
    в 0, принимает значение установленного в системе для текущей
    локали пользователя.

[*] Все возможные текстовые сообщения перенесены из кода плагина в
    языковые файлы

[*] Текст пароля убирается из заголовка панели, лога и окна команд
    Заменяется текстом "*hidden*".

[*] Изменен алгоритм сортировки записей: сортирует по имени сервера, имени
    пользователя, описанию - по очереди (т.е. если имена сервера одинаковые,
    то сравнивает имена пользователей, и т.д.)

[*] Добавлены ".." на панель хостов

[*] Изменен алгоритм отображения цифр при копировании: диалог копирования
    теперь имеет размер, зависящий от размера максимального числа, в
    результате предел отображаемых значений - до 2^64

[*] Нет перекрывающегося сообщения "Scanning...", которое рисует
    FAR при подсчете размеров файлов для копирования

[*] Статус текущего сканирования теперь отображается в нижней части окна
    FTP команд

[*] В информации версии плагина FileVersion установлена в версию плагина,
    ProductVersion в номер версии FAR-а

[*] Цифры отображающие текущий CPS компрессируются до Мбайт и Кбайт

[*] Показывает Idle на заголовке или на экране (TechInfoFTP.Rus.txt#19)

[*] Сохраняет режим панели плагина и восстанавливает его при следующей
    активации панели

[*] Сохраняет все установки при подтверждении в диалоге конфигурации

[*] показывает правильное сообщение об ошибке если не удалось инициализировать
    библиотеку socket

ProcList
~~~~~~~~
[+] Можно просматривать список процессов на удаленных машинах (F6),
    если на них работает NT или Windows 2000. Для этого у вас должны быть
    права администратора. Для просмотра списка процессов из-под Windows

[+] В режиме просмотра распознается больше дополнительных стилей
    окон.
[+] Под NT или Windows 2000 можно убивать службы (сервисы) и другие

    существует, только если у вас права администратора. БУДЬТЕ ОСТОРОЖНЫ!
