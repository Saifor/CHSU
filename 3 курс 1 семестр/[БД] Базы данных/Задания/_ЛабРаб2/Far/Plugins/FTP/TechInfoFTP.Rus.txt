  Формат описания файла:

      [Имя ключа]: Полный путь ключа в реестре для копии FAR-а, использующей
                   основного пользователя (см. описание ключа командной строки
                   "/u" в документации FAR)
   Логический тип: Логический тип ключа.
                   Возможные значения логического типа:
                     bool   - Бинарное значение, которое может принимать
                              только два значение: "истина" или "ложь"
                              ("да" или "нет").
                              Числовое значение из реестра интерпретируется
                              как "истина", если содержит ненулевое значение,
                              и как "ложь", если содержит ноль
                     int    - Число
                     String - Строка
                     char   - Один символ.
                              Числовое значение ключа реестра будет
                              интерпретировано как код символа в таблице OEM.
    Тип в реестре: Тип, который нужно использовать, создавая значения в
                   реестре.
     Имя значения: Имя ключа в реестре.
         Значение: Значение "по умолчанию", т.е., то значение, которое плагин
                   будет использовать для этого ключа, если ключ отсутствует
                   в реестре.
       Назначение: Описание/назначение ключа в плагине.
               !!: Предупреждение/замечание по использованию
------------------------------------------------------------------------

 1.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\DigitDelimit]
   Логический тип: bool
    Тип в реестре: DWORD
     Имя значения: DigitDelimit
         Значение: TRUE
       Назначение: Разделять ли тысячные порядки при выводе чисел в диалоге
                   копирования.
                   К примеру: если DigitDelimit установлен в "истину", и
                   разделитель установлен в '.', то число "3000" будет
                   отображено как "3.000"

 2.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\DigitDelimiter]
   Логический тип: char
    Тип в реестре: DWORD (code of OEM character)
     Имя значения: DigitDelimiter
         Значение: 0
       Назначение: Тысячный разделитель.
                   Используется совместно с DigitDelimit.
                   Может принимать значение символа разделителя или нулевое
                   значение.
                   Если разделитель задан нулем, то будет использоваться
                   текущий системный разделитель.
               !!: Обычно системный разделитель в Windows для System Locale
                   "Русская" задан так, что символ, который будет отображаться
                   плагином, будет не "," (который отображается в графических
                   диалогах Windows), а " " (пробел).

 3.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\WaitTimeout]
   Логический тип: int
    Тип в реестре: DWORD
     Имя значения: WaitTimeout
         Значение: 300 (секунд)
       Назначение: Максимальный период ожидания минимального пакета данных в
                   секундах.
                   Если в процессе ожидания данных или команд от сервера или
                   передачи их на сервер прошло больше секунд, чем указанное
                   значение, то операция будет отменена как ошибочная, со
                   статусом ошибки, установленным в
                     "Период ожидания завершения операции истек"
               !!: 1. Этот период используется не для глобальных операций
                   (таких, как "послать файл" или "прочитать какталог"), а для
                   "минимальных" операций обмена данными с сервером.
                     Обычно любая глобальная операция состоит из множества
                   минимальных.
                   2. После каждого успешного совершения минимальной операции
                   счетчик секунд ожидания сбрасывается в ноль.

 4.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\AskAbort]
   Логический тип: BOOL
    Тип в реестре: DWORD
     Имя значения: AskAbort
         Значение: TRUE
       Назначение: Спрашивать подтверждение на прерывание текущей операции,
                   если во время этой операции пользователь нажал на ESC.
               !!: Если этот параметр отключен, то плагин будет считать, что
                   прерывание подтверждено.

 5.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\CmdLength]
   Логический тип: int
    Тип в реестре: DWORD
     Имя значения: CmdLength
         Значение: 7
       Назначение: Количество строк, которые будут отображаться в окне
                   выполнения команд плагина.
               !!: Если Вы используете FAR версии меньше, чем
                     1.70 alpha 4 (build 1000)
                   то нельзя задавать значения более 8, т.к. FAR содержит
                   ошибку вывода, исправленную в более поздних версиях.

 6.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\CmdLine]
   Логический тип: int
    Тип в реестре: DWORD
     Имя значения: CmdLine
         Значение: 70
       Назначение: Длина окна выполнения команд.
                   Это значение определяет ширину окна команд.

 7.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\WaitIdle]
   Логический тип: int
    Тип в реестре: DWORD
     Имя значения: WaitIdle
         Значение: 0
       Назначение: Пауза при выполнении операций связи в миллисекундах.
                   Если операции связи выполнять без пауз, то все время
                   процессора будет занято их выполнением (даже если это
                   операции ожидания).
                   Однако, если задавать слишком большое значение, то могут
                   возникнуть ошибки связи из-за того, что сервер не получил
                   ответ на свой запрос вовремя.
                   Значение, установленное в ноль, плагин интерпретирует как
                   команду "работать без пауз"
               !!: 1. Если Вас интересует максимальная производительность
                   плагина (например, Вы используете его интерактивно), то
                   установите это значение в ноль или в небольшую (10-20)
                   величину.
                     Если Вам хочется, чтобы работа плагина была незаметна для
                   окружающих программ (например, Вы используете плагин для
                   фоновых операций), то подберите это значение так, чтобы
                   загрузка процессора была такой, как Вы хотите.
                   Рекомендуется параллельно с FAR-ом запустить одну из
                   программ измерения нагрузки процессора (TaskManager под
                   WinNT, или сторонние средства под Win9X) и, изменяя эту
                   величину, добиваться необходимой загрузки процессора.
               !!: Для тех, кто использует Win9X, я рекомендую воспользоваться
                   программой TaskInfo:
                     http://www.iarsn.com/downloads

 8.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\CmdLogFile]
   Логический тип: String
    Тип в реестре: STRING
     Имя значения: CmdLogFile
         Значение: ""
       Назначение: Имя файла, в который будут записываться команды, выполняемые
                   плагином, и ответы сервера, а также текстовые результаты
                   выполнения команд.
                   Если имя не задано, или задано как пустая строка, то команды
                   не будут записываться на диск.
                   Если имя, заданное в этом ключе, не начинается с буквы
                   диска, то оно интерпретируется как путь и имя относительно
                   каталога, в котором расположен плагин.
                   Пример полного пути: "C:\Logs\FARFtp.log"
               !!: Каталог, в который будет записан лог, должен существовать.
               !!: Если Вы считаете, что в процессе работы плагина происходят
                   ошибки, то вместе с описанием этих ошибок нужно посылать ту
                   часть файла команд, которая касается неправильных действий.

 9.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\CmdLogLimit]
   Логический тип: int
    Тип в реестре: DWORD
     Имя значения: CmdLogLimit
         Значение: 100 (100.000 bytes)
       Назначение: Максимальный размер файла команд в килобайтах (1000 байт).
                   Если файл достиг или превысил указанный размер, то он будет
                   обнулен.
                   Если Вы не хотите ограничивать размер файла команд, то
                   установите это значение в ноль.

10.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\IOBuffSize]
   Логический тип: int
    Тип в реестре: DWORD
     Имя значения: IOBuffSize
         Значение: 10.000 (10.000 bytes)
       Назначение: Размер буфера, используемого в операциях чтения/записи
                   файлов с/на FTP.
               !!: Диалог копирования обновляется не раньше, чем полный буфер
                   будет прочитан/записан, поэтому если вы увеличиваете это
                   значение, то уменьшается точность отображения процесса в
                   диалоге копирования.

11.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\BeepTimeout]
   Логический тип: int
    Тип в реестре: DWORD
     Имя значения: BeepTimeout
         Значение: 30 (sec)
       Назначение: Время, не меньше которого должна потребовать операция, чтобы
                   она считалась длительной (в секундах).
                   После того, как завершается длительная операция, будет выдан
                   стандартный звуковой сигнал Windows.
                   Если это значение установлено в ноль, то уведомление об
                   окончании длительных операций будет отключено.

12.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\ShowIdle]
   Логический тип: bool
    Тип в реестре: DWORD
     Имя значения: ShowIdle
         Значение: TRUE
       Назначение: Показывать состояние ожидания.
                   Если это значение установлено в "истина", то в верхнем левом
                   углу экрана FAR-а будет выводиться надпись:
                     "Idle(XX): YY%"
                   означающая, что плагин ожидает приема или передачи данных.
                   В этой надписи:
                     XX - время ожидания (в секундах), которое прошло с
                          момента начала ожидания
                     YY - процент прошедшего времени по сравнению с
                          установленным временем ожидания (WaitTimeout)
               !!: 1. Из-за специфики API FAR-а это время не может быть удалено
                   с экрана с восстановлением содержимого, которое было до
                   вывода этого сообщения. Поэтому, когда ожидание завершилось,
                   в том же месте экрана будет выведена надпись из файла
                   языков:
                     "-- Working...--" - для английского языка
                   2. Как только время достигнет установленного в "WaitTimeout"
                   значения, операция будет прервана (см описание п.3)
                   3. Операция может быть прервана и до того момента, как
                   период ожидания достигнет заданной величины. Это может
                   произойти из-за следующих причин:
                     - время ожидания сервера, с которым соединен плагин,
                     установлено в меньшую величину
                     - время ожидания, установленное в Вашей операционной
                     системе, установлено в меньшую величину
                     - произошла одна из возможных стандартных ошибок
                     приема/передачи данных

13.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\IdleColor]
   Логический тип: int
    Тип в реестре: DWORD
     Имя значения: IdleColor
         Значение: FAR_COLOR(fccCYAN,fccBLUE)
       Назначение: Цвет надписи "ShowIdle" (см.), который будет использоваться
                   для вывода сообщения.
                   Цвет является значением от 0 до 255, определяющим
                   стандартные цвета консоли.
               !!: О том, как формировать значение цвета из цвета букв и цвета
                   фона, см. техническую документацию API о выводе на консоль
                   Windows.
                   Гораздо проще воспользоваться для получения значения одним
                   из многих плагинов к FAR-у, которые либо предназначены для
                   формирования этого значения:
                     "Color selector table", "TrueColor", ...
                   либо отображают это значение в диалоге выбора цвета:
                     "Crest II", ...

14.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\TimeoutRetry]
   Логический тип: BOOL
    Тип в реестре: DWORD
     Имя значения: TimeoutRetry
         Значение: FALSE
       Назначение: Повторить выполнение команды, если команда была прервана
                   из-за того, что либо произошло превышение установленного в
                   плагине предела ожидания (см. WaitTimeout), либо код
                   стандартной ошибки выполнения команды установлен в "период
                   ожидания истек"
               !!: 1. Команда предназначена для использования только при
                   чтении/записи файла с/на FTP
                   2. Если хотя бы один раз произошло автоматическое повторение
                   команды, то в диалоге копирования и в заголовке окна FAR-а
                   будет выводиться общее количество произошедших повторов
                   3. Если в процессе выполнения команды произошла одна из
                   других ошибок (к примеру: "файл существует", "файл только
                   для чтения", и т.д.), то будет выведено стандартное окно,
                   соответствующее ошибке, и плагин будет ожидать реакции
                   пользователя.
                     В связи с этим, если Вы хотите производить операции в
                   автоматическом режиме (к примеру тогда, когда Вас нет за
                   компьютером), следует заранее устранить источник подобных
                   ошибок.

15.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\RetryCount]
   Логический тип: int
    Тип в реестре: DWORD
     Имя значения: RetryCount
         Значение: 0
       Назначение: Максимальное количество автоматических повторов.
                   Используется совместно со значением "TimeoutRetry".
                   Если количество повторов установлено в ноль или не
                   установлено вообще, то плагин не будет ограничивать
                   количество автоматических повторов.
                   Если значение установлено, и количество повторов достигло
                   этой величины, то будет выдан стандартный интерактивный
                   диалог сообщения об ошибке.
               !!: Счетчик количества повторов сбрасывается после каждого
                   обработанного файла.
                   Текущее значение количества автоматических повторов
                   отображается в заголовках диалога копирования и окна FAR-а.

17.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\PluginColumnMode]
   Логический тип: int
    Тип в реестре: DWORD
     Имя значения: PluginColumnMode
         Значение: 2
       Назначение: Номер режима панели плагина, который будет устанавливаться
                   в начале работы с записями серверов.
               !!: 1. Плагин определяет три собственных режима для номеров:
                       1, 2 и 3.
                   2. Этот номер будет использоваться только при начальном
                   открытии панели плагина. В процессе работы с панелью номер
                   запоминается, и в будущем (при открытии панели записей)
                   будет использоваться номер, который был использован в
                   последний раз.
                   3. Номер режима индивидуален для плагина.
                   Если плагин закрылся, то при следующем использовании опять
                   будет использован номер режима, заданного значением
                   "PluginColumnMode"
                   4. Номера должны задаваться в соответствии с цифрами,
                   к которым они относятся. Т.е.:
                     0 - для режима, устанавливающегося нажатием Ctrl+0
                     1 - для режима, устанавливающегося нажатием Ctrl+1
                   и т.д.

   Логический тип: bool
    Тип в реестре: DWORD

                   общего скопированного обьема.

19.[HKEY_CURRENT_USER\Software\Far\plugins\FTP\IdleMode]
   Логический тип: int
    Тип в реестре: DWORD
     Имя значения: IdleMode
         Значение: 1
       Назначение: Определяет способ показа Idle информации.
                   Если установлено в 1 - показывает на экране ФАРа;
                   Если установлено в 2 - показывает в заголовке консоли;
                   Если установлено в 1 - показывает и там и там;
